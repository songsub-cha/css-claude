# Domain Expert Expansion — Implementation Plan (Epic Skeleton)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Plan level:** SKELETON (CSS epic). Coarse tasks + batches + single-domain tags + file targets. **No per-step code here** — detailed per-Phase plans (full RED/GREEN) are generated by `/css:phase` → `/css:plan --session <child>`.

**Goal:** CSS 도메인 전문가를 7→10종으로 확대(node-backend·spring-backend·ml-engineer 신설)하고 db/ui/infra를 대표 스택으로 강화하며, 라우팅(executor/reviewer)·문서(README)·정합성 테스트를 동기화한다.

**Architecture:** 신설/강화 대상은 모두 `agents/*.md` 시스템 프롬프트(→ `css-prompt-engineer` 도메인). 라우팅 파일(`executor.md`/`reviewer.md`)도 프롬프트 마크다운(→ prompt-engineer). Python 정합성 도구(`tools/agent_registry/`)와 README 문서는 도메인 특화가 아니므로 executor-direct. `agent_registry` 정합성 테스트(agents ↔ dispatch table ↔ README)가 prose 변경의 회귀 가드이자 prompt 태스크의 RED 하니스.

**Tech Stack:** Markdown 에이전트 프롬프트, Python 3.12 + `unittest`(기존 `tools/css_schema` 패턴), 정규식 기반 마크다운 파서.

**Spec:** `docs/superpowers/specs/2026-06-01-domain-expert-expansion-design.md`

## Dependency / Batch 요약 (위상 순서)

```
Batch 1: T1 (guard)
Batch 2: T2 T3 T4 T5 T6 T7 T8        (병렬; 서로 다른 파일)
Batch 3: T9 T10 T11                  (각자 전용 파일; T2–T4 의존)

```

**Phase 분할 힌트 (`/css:phase`용):**
- Phase 1 = Batch 1 (정합성 가드).
- Phase 2 = Batch 2 (에이전트 신설·강화).
- Phase 3 = Batch 3 + 4 (라우팅/문서 동기화 + 검증).
→ Phase 2는 Phase 1(가드) 의존, Phase 3은 Phase 2 의존. 임계 미만이면 단일 Epic 유지 가능.
